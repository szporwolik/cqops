# Руководство пользователя CQOps

CQOps — это быстрый радиолюбительский логгер, ориентированный на работу в терминале. Он предназначен для операторов, которым нужен надежный ввод связей с клавиатуры при низкой нагрузке на систему. CQOps разработан для использования в шэке, при выездной работе, на клубных радиостанциях, во время field day, а также на устройствах класса Raspberry Pi и старых ноутбуках.

CQOps всегда сначала сохраняет QSO локально. Интеграции, использующие интернет, являются необязательными.

<a id="what-cqops-is"></a>
## Что такое CQOps

CQOps построен вокруг быстрого ввода QSO, локального хранения данных в первую очередь и практичной работы в полевых условиях.

<a id="main-ideas"></a>
### Основные принципы

- **Работа прежде всего в терминале** — оптимизировано для управления с клавиатуры.
- **Автономная регистрация связей** — локальная запись QSO работает без доступа к интернету. В dashboard встроена карта мира, которая полностью работает офлайн.
- **Низкая нагрузка** — подходит для систем класса Raspberry Pi, старых ноутбуков и общих компьютеров радиостанции.
- **Портативная архитектура** — распространяется как один исполняемый файл Go.
- **Несколько аппаратных журналов** — удобно для личных, выездных, соревновательных и клубных журналов.
- **Несколько операторов** — удобно для работы по принципу hot-seat и на общей клубной радиостанции.
- **Несколько трансиверов** — каждый preset трансивера может хранить собственные настройки backend и WSJT-X.
- **Необязательные интеграции** — callbook с несколькими провайдерами (QRZ.com, HamQTH, QRZ.RU, Callook.info), Wavelog, DX Cluster, PSK Reporter, GPS, APRS, управление трансивером, управление поворотным устройством, солнечные данные и браузерный dashboard CQOps Live.

Для локального ведения журнала доступ к интернету не требуется. В режиме `--offline` сетевые функции пропускаются.

<a id="who-cqops-is-for"></a>
### Для кого предназначен CQOps

CQOps хорошо подходит для:

- операторов, работающих выездным образом,
- активаторов SOTA и POTA,
- клубных радиостанций,
- команд field day,
- операторов, предпочитающих работу в терминале,
- радиостанций, где требуется быстро переключаться между операторами, аппаратными журналами или трансиверами.

CQOps не предназначен для замены всех возможностей полноценного настольного логгера или веб-платформы аппаратного журнала. Основной акцент сделан на быстром вводе в терминале, полевой работе, автономном режиме и сценариях общей радиостанции.

<a id="club-and-shared-station-use"></a>
### Использование в клубе и на общей радиостанции

CQOps создавался с учетом потребностей радиоклубов. Активный оператор всегда виден в строке состояния — **одного взгляда** достаточно, чтобы понять, кто сейчас работает. Оператор переключается одним сочетанием клавиш (`Ctrl+O`), а изменение применяется немедленно: позывной и имя оператора записываются во все последующие QSO. Не требуется выход из учетной записи, ввод пароля или остановка работы.

Аппаратные журналы, presets трансиверов и соревнования переключаются аналогичным образом — `Ctrl+L`, `Ctrl+R`, `Ctrl+C`. На клубной радиостанции со сменяющимися операторами, несколькими трансиверами и несколькими активными соревнованиями контекст можно изменить менее чем за секунду, не используя мышь.

Во время field day и публичных мероприятий **CQOps Live dashboard** может выводить на большой экран карту в реальном времени, поток QSO и статистику. Посетители и члены клуба смогут следить за работой станции, не собираясь вокруг терминала оператора. Достаточно включить интеграцию HTTP Server и открыть dashboard на любом устройстве с веб-браузером.

<a id="first-launch"></a>
## Первый запуск

При первом запуске CQOps открывает мастер настройки. Для локального ведения журнала обязательны только основные сведения о радиостанции. Сетевые интеграции можно пропустить и настроить позже.

<a id="wizard-pages"></a>
### Страницы мастера

| Страница | Что настраивается |
|---|---|
| Station & Logbook | Начальный аппаратный журнал, позывной станции, оператор, grid locator, необязательные референции и зоны, URL/API/ID station profile Wavelog |
| Rig | Preset трансивера, модель, антенна, мощность, backend, необязательное поворотное устройство и необязательные параметры UDP WSJT-X |
| Integrations | Параметры поиска в callbook (QRZ.com, HamQTH, QRZ.RU, Callook.info) |
| General | Часовой пояс IANA |
| Summary | Проверка и сохранение |

Поддерживаемые backend трансивера:

- None,
- flrig,
- Hamlib `rigctld`.

<a id="wizard-navigation"></a>
### Навигация по мастеру

| Клавиша | Действие |
|---|---|
| Ctrl+S | Проверить и продолжить; на странице Summary — сохранить и запустить CQOps |
| Esc | Назад |
| F10 | Выход |
| Tab / Shift+Tab | Перемещение между полями |
| Space | Переключение флажков |

Впоследствии параметры мастера можно изменить с помощью **F9**.

<a id="log-your-first-qso"></a>
## Регистрация первого QSO

1. Запустите CQOps:

   ```bash
   cqops
   ```

2. Завершите мастер настройки, указав как минимум свой позывной и grid locator.

3. Откройте форму QSO клавишей **F1**.

4. Введите позывной корреспондента. CQOps автоматически переводит позывные в верхний регистр.

5. Заполните остальные поля. Если активный трансивер подключен через flrig или Hamlib, CQOps может автоматически заполнить частоту, диапазон, вид излучения и подвид.

6. Нажмите **Enter**, чтобы сохранить.

7. Если появится предупреждение **DUPE!**, снова нажмите **Enter**, чтобы все равно сохранить запись, или **Esc**, чтобы отменить.

Сохраненный QSO немедленно появится в таблице Recent QSOs.

- активный трансивер,

| Цвет | Значение |
|---|---|
| Белый/по умолчанию | Подключено или активно |
| Желтый | Отключено, выполняется подключение или ожидаемо недоступно |
| Красный | Ошибка или отсутствие подключения |
| Акцентный цвет + полужирный | WSJT-X выполняет передачу |

<a id="common-workflows"></a>
## Типовые сценарии работы

<a id="portable-sota-or-pota-operation"></a>
### Выездная работа, SOTA или POTA

Перед выездом:

1. Один раз запустите CQOps при наличии доступа к интернету.
2. Позвольте CQOps загрузить или обновить кэшированные данные, например солнечные данные, данные REF и префиксы DXCC.
3. Убедитесь, что на панели Solar отображаются данные.
4. Убедитесь, что поиск REF по **F6** возвращает результаты.

В полевых условиях:

1. Запустите CQOps в автономном режиме:

   ```bash
   cqops --offline
   ```

2. Регистрируйте связи обычным образом. QSO сохраняются локально.
3. После восстановления доступа к интернету откройте **F8** и нажмите **w**, чтобы отправить в Wavelog еще не переданные QSO.

<a id="shared-club-station-and-hot-seat-logging"></a>
### Общая клубная радиостанция и работа hot-seat

1. Откройте **F9 → Operators**.
2. Нажмите **Ins**, чтобы добавить профили операторов.
3. В форме QSO нажмите **Ctrl+O**, чтобы переключить активного оператора.
4. Перед сохранением проверьте активного оператора в строке состояния.
5. Используйте **Retain**, если нескольким операторам необходимо регистрировать похожие связи без повторного заполнения всей формы.

Активный оператор сохраняется в поле ADIF `OPERATOR`.

<a id="personal-and-club-logbooks"></a>
### Личные и клубные аппаратные журналы

1. Откройте **F9 → Logbooks**.
2. Нажмите **Ins**, чтобы создать каждый аппаратный журнал.
3. В форме QSO нажмите **Ctrl+L**, чтобы переключить активный аппаратный журнал.
4. Перед сохранением проверьте активный журнал в строке состояния.

Каждый аппаратный журнал может хранить собственные сведения о станции, параметры Wavelog, настройки соревнований и операторов.

<a id="multiple-rigs"></a>
### Несколько трансиверов

1. Откройте **F9 → Rigs**.
2. Нажмите **Ins**, чтобы создать presets трансиверов.
3. Выберите backend: None, flrig или Hamlib.
4. В форме QSO нажмите **Ctrl+R**, чтобы переключить активный трансивер.

Preset трансивера может содержать backend, модель, антенну, мощность, параметры поворотного устройства и настройки UDP WSJT-X.

<a id="wsjt-x-digital-operation"></a>
### Цифровая работа с WSJT-X

Если интеграция WSJT-X по UDP включена, CQOps может принимать сообщения ADIF от WSJT-X и автоматически регистрировать завершенные цифровые QSO.

Автоматически зарегистрированные QSO:

- сохраняются в активный аппаратный журнал,
- немедленно появляются в Recent QSOs,
- пропускают дубликаты,
- получают ID активного соревнования,
- могут автоматически отправляться в Wavelog, если он настроен и доступен.

Если оператор, указанный WSJT-X, не совпадает с активным оператором в CQOps, программа отображает предупреждение.

Перед длительными цифровыми сессиями проверьте:

- активный аппаратный журнал,
- активного оператора,
- активное соревнование,
- метку состояния WSJT-X.

<a id="wavelog-sync"></a>
### Синхронизация с Wavelog

CQOps всегда сначала сохраняет QSO локально. Синхронизация с Wavelog необязательна.

| Действие | Где | Сочетание | Примечания |
|---|---|---|---|
| Отправить неотправленные QSO | Logbook Editor | `w` | Отправка пакетами по 50 |
| Загрузить из Wavelog | Logbook Editor | `Ctrl+W` | Инкрементальная загрузка с использованием `last_fetched_id` |

Для каждого QSO отслеживается состояние отправки:

- не отправлено,
- отправлено,
- ошибка.

При ошибке отправки QSO остается в локальном аппаратном журнале, и попытку можно повторить позже. Полная очистка журнала сбрасывает fetch ID на `0`, что позволяет выполнить полную повторную загрузку.

<a id="qso-logging"></a>
## Регистрация QSO

Форма QSO — основной экран ввода связей. Откройте ее клавишей **F1**.

CQOps может заполнять поля из следующих источников:

| Источник | Поля |
|---|---|
| flrig / Hamlib | Частота, Freq RX при split, вид излучения, подвид |
| Callbook (QRZ.com / HamQTH / QRZ.RU / Callook.info) | Имя, QTH, grid, страна, зона CQ, зона ITU, DXCC, континент, фото |

| Экспорт ADIF | Ctrl+E |

Форма QSO содержит отдельные поля **Mode** и **Submode**. Оба поля можно переключать клавишами **PgUp/PgDn**.

<a id="contests"></a>
## Соревнования

CQOps содержит облегченную панель для регистрации связей в соревнованиях, предназначенную для **неформального участия**. Она не заменяет специализированные соревновательные логгеры, такие как N1MM, Win-Test или TR4W. Для серьезной работы в категориях multi-op, multi-radio или assisted используйте специализированный логгер. CQOps подходит, когда вы хотите провести несколько связей и раздать очки, из интереса проследить за темпом работы или зарегистрировать несколько соревновательных QSO во время активации SOTA/POTA, не покидая повседневный логгер.

<a id="setting-up-a-contest"></a>
### Настройка соревнования

Создайте или настройте соревнование в Logbook Editor с помощью **Ins**.

Конфигурация соревнования включает:

- название соревнования,
- дату,
- идентификатор соревнования ADIF,
- шаблоны контрольных номеров.

<a id="template-markers"></a>
#### Маркеры шаблонов

| Маркер | Заменяется на |
|---|---|
| `@rst` | Переданный или принятый RST |
| `@serial` | Автоматически увеличиваемый порядковый номер |
| `@cqz` | Зона CQ DX-станции |
| `@mycqz` | Ваша зона CQ |
| `@itu` | Зона ITU DX-станции |
| `@myitu` | Ваша зона ITU |
| `@grid` | Grid square DX-станции |
| `@mygrid` | Ваш grid square |

Нажмите **Ctrl+C**, чтобы переключить активное соревнование, или выберите его в меню Contest (**F7**). Поля контрольных номеров автоматически появятся в форме QSO, а порядковые номера будут увеличиваться автоматически.

<a id="bottom-status-bar"></a>
### Нижняя строка состояния

При активном соревновании в нижней строке отображается сводка в реальном времени:

```
 IARU-HF · IARU HF   45 QSOs   Started 16:13   Last 14:04 ago   Next #45   On 2:41
```

| Поле | Значение |
|-------|---------|
| `IARU-HF` | Идентификатор соревнования ADIF, предназначенный для машинной обработки |
| `· IARU HF` | Отображаемое название соревнования — показывается, если отличается от ID |
| `45 QSOs` | Общее количество QSO в текущей сессии соревнования |
| `Started 16:13` | Время первого соревновательного QSO за текущий день |
| `Last 14:04 ago` | Время, прошедшее с последнего соревновательного QSO |
| `Next #45` | Порядковый номер, который будет передан в следующем QSO |
| `On 2:41` | Общее эфирное время — сумма промежутков между QSO длительностью менее 30 минут |

Поле "Started" скрывается в узких терминалах, имеющих менее 120 столбцов. Название соревнования и эфирное время скрываются при ширине менее 100 столбцов.

<a id="contest-statistics-panel"></a>
### Панель статистики соревнования

Если соревнование активно и терминал достаточно широкий, справа от формы QSO появляется компактная панель статистики с желтой рамкой:

```
╭──────────────────────────────────╮
│  Rate     2/h   --/h             │
│  Count 60m   0  hr   0           │
│  Peak  1m120 10m 54 60m 29       │
│  Avg      8/h  Sess 5:36         │
│  QSO/min  last 60m  max 1        │
│                                  │
│                                  │
│                                  │
│                                  │
│  -60m                       now  │
╰──────────────────────────────────╯
```

| Строка | Поле | Значение |
|-----|-------|---------|
| **Rate** | `2/h` | Темп по последним **10 QSO** — кратковременная скорость работы |
| | `--/h` | Темп по последним **100 QSO** — отображает `--`, пока не зарегистрировано 100 QSO |
| **Count** | `60m 0` | Количество QSO за последние 60 минут |
| | `hr 0` | Количество QSO за текущий час, начиная с `:00` |
| **Peak** | `1m120` | Лучший темп за 1 минуту: 120/ч = 2 QSO за эту минуту |
| | `10m 54` | Лучшее скользящее окно 10 минут: средний темп 54/ч |
| | `60m 29` | Лучшее скользящее окно 60 минут: средний темп 29/ч |
| **Avg** | `8/h` | Средний темп сессии — общее число QSO ÷ часы с момента первого QSO |
| | `Sess 5:36` | Общая длительность сессии от первого до последнего QSO, в формате H:MM или только минуты |
| **Chart** | `max 1` | В самую активную минуту проведен 1 QSO. Столбики показывают число QSO в минуту |
| | `-60m…now` | Левая граница = 60 минут назад, правая граница = сейчас |

График использует блочные символы Unicode (`█`), масштабированные до четырех строк вертикальных столбиков. Для пиковых значений суффикс `/h` опускается, поскольку "Peak" уже подразумевает значение в час. Во всех длительностях секунды опущены — при обновлении раз в минуту они создают только лишний шум.

<a id="contest-adif-export"></a>
### Экспорт ADIF для соревнования

Чтобы отправить соревновательный журнал, откройте **Logbook Editor** (`Ctrl+E`), пока соревнование активно. Если применен фильтр соревнования, диалог экспорта ADIF предложит экспортировать **только QSO, относящиеся к активному соревнованию**. Будет создан соответствующий стандарту файл ADIF 3.1.7 с сохраненными полями контрольных номеров, порядковыми номерами и идентификатором соревнования ADIF. Файл готов для отправки роботу организатора или в систему проверки журналов.

<a id="contest-mode-behavior"></a>
### Поведение режима соревнования

При активном соревновании:

- в форме QSO отображаются поля контрольных номеров,
- порядковые номера увеличиваются автоматически,
- Recent QSOs можно отфильтровать до QSO соревнования,
- при экспорте ADIF сохраняется `CONTEST_ID`,
- форма QSO, панель соревнования и панель Solar получают желтую рамку для визуального выделения,
- споты DXC проверяются на дубликаты по всем QSO соревнования, а не только за текущий день.

## Избранное, референции и планы диапазонов

<a id="favorites"></a>
### Избранное

Избранное хранит presets частоты, вида связи и диапазона в трех слотах — этого достаточно для наиболее часто используемых вызывных частот. Сочетания используют `Alt`, чтобы избежать конфликтов со стандартными клавишами редактирования терминала, и надежно работают в терминалах всех типов.

| Сочетание | Действие |
|---|---|
| Alt+Ins / Alt+Home / Alt+PgUp | Вызвать избранное из слота 1, 2 или 3 |
| Alt+Shift+Ins / Alt+Shift+Home / Alt+Shift+PgUp | Сохранить текущую частоту, вид связи и диапазон в слот 1, 2 или 3 |

Избранное хранится в конфигурации и является общим для всех аппаратных журналов.

Пример:

1. Введите `145.55`.
2. Установите Mode в `FM`.
3. Установите Band в `2m`.
4. Нажмите **Alt+Shift+Ins**, чтобы сохранить в слот 1.
5. Позже нажмите **Alt+Ins**, чтобы вызвать preset.

<a id="ref-lookup"></a>
### Поиск REF

Откройте REF Lookup клавишей **F6**.

Поиск выполняется по:

- SOTA,
- POTA,
- WWFF,
- IOTA.

Можно искать по префиксу, названию или обозначению референции. Выбранные референции можно перенести в форму QSO.

<a id="band-plan-browser"></a>
### Просмотр планов диапазонов

Откройте Band Plan Browser клавишей **F7**.

Он предоставляет быстрый доступ к:

- любительским диапазонам,
- участкам VHF/UHF,
- CB,
- PMR446,
- presets радиовещательных частот,
- Portable — распространенным частотам выездной и полевой работы, включая SOTA, POTA и вызывные каналы.

Выбранную частоту можно использовать для настройки активного трансивера. Данные плана диапазонов также можно экспортировать в Markdown.

<a id="integrations"></a>
## Интеграции

Все интеграции необязательны. Локальное ведение журнала работает без них.

<a id="callbook-qrzcom-hamqth-callookinfo"></a>
### Callbook (QRZ.com, HamQTH, QRZ.RU, Callook.info)

CQOps поддерживает несколько провайдеров callbook с каскадным поиском по приоритету.
Когда в форме QSO нажимается **Ins**, провайдеры опрашиваются по порядку, пока один из них не вернет результат:

1. **QRZ.com** — требует доступа к интернету и подписки QRZ XML. Предоставляет наиболее полные данные.
2. **HamQTH** — бесплатный глобальный сервис с хорошим покрытием. Требует бесплатной учетной записи.
3. **QRZ.RU** — бесплатный сервис, ориентированный на Россию и соседние страны. Требует API-логин. Возвращает имя, QTH, grid, широту/долготу, класс, статус LoTW/eQSL и фото.
4. **Callook.info** — бесплатный сервис, ориентированный на США. Учетная запись не требуется; обеспечивает быстрый поиск по данным FCC.

Если провайдеры с более высоким приоритетом недоступны или отключены, используется следующий провайдер.
Если включен параметр **Base call fallback** (по умолчанию включен), CQOps также проверяет базовый позывной без префикса или суффикса, если полный позывной не найден.

Провайдеры включаются и настраиваются в **F9 → Callbook**.

В форме QSO нажмите **Ins**, чтобы заполнить поля callbook, включая:

- имя,
- QTH,
- grid,
- страну,
- зоны CQ/ITU,
- DXCC,
- континент.

В Partner view на **F2** может отображаться фотография оператора, если она доступна.

> ⚠️ **Экспериментальная функция.** Для показа фотографии может использоваться
> графический протокол терминала Kitty. Требуется совместимый терминал: Kitty,
> Ghostty или WezTerm. Включите функцию в **F9 → General → Kitty Graphics**.
> В обычных терминалах и SSH-сеансах без передачи графики вместо фотографии
> будет использоваться изображение из символов.

<a id="wavelog"></a>
### Wavelog

Интеграция с Wavelog поддерживает:

- отправку данных,
- инкрементальную загрузку,
- поиск состояния worked/confirmed.

Wavelog настраивается отдельно для активного аппаратного журнала с помощью:

- URL,
- API key,
- station profile ID.

CQOps всегда сначала сохраняет QSO локально. Ошибка отправки в Wavelog не удаляет локальные данные.

<a id="flrig"></a>
### flrig

Интеграция с flrig использует XML-RPC поверх HTTP.

Конечная точка по умолчанию:

```text
localhost:12345
```

CQOps может считывать:

- частоту,
- вид излучения,
- мощность.

При работе split VFO A соответствует Frequency, а VFO B — Freq RX.

<a id="hamlib--rigctld"></a>
### Hamlib / rigctld

Управление трансивером через Hamlib использует TCP-демон `rigctld`.

В зависимости от трансивера и возможностей backend CQOps может запрашивать:

- частоту,
- вид излучения,
- VFO,
- split,
- мощность.

По возможности CQOps корректно обрабатывает отсутствие поддержки имен VFO.

<a id="hamlib-rotator--rotctld"></a>
### Поворотное устройство Hamlib / rotctld

> ⚠️ **Экспериментальная функция.** Управление поворотным устройством является
> экспериментальным. Перед работой всегда проверяйте физические пределы антенны.
> Будьте готовы немедленно остановить движение сочетанием **Alt+/**. Используйте
> функцию осторожно: неправильная конфигурация может повредить поворотное
> устройство или антенну.

Для управления поворотным устройством используется Hamlib `rotctld`.

CQOps поддерживает:

- азимут,
- угол места,
- команды остановки.

| Сочетание | Действие |
|---|---|
| Alt+, | Изменить азимут на −5° |
| Alt+. | Изменить азимут на +5° |
| Alt+; | Изменить угол места на +5° |
| Alt+' | Изменить угол места на −5° |
| Alt+\ | Направить антенну по рассчитанному азимуту трассы |
| Alt+/ | Остановить поворотное устройство |

<a id="wsjt-x"></a>
### WSJT-X

Интеграция с WSJT-X использует UDP-сообщения WSJT-X. CQOps разбирает сообщения ADIF и может автоматически регистрировать завершенные QSO.

Во время передачи WSJT-X метка трансивера окрашивается акцентным цветом. Если оператор, указанный WSJT-X, не совпадает с активным оператором, CQOps отображает предупреждение.

<a id="gps"></a>
### GPS

CQOps может получать координаты от GPS-приемника и использовать их как grid locator станции — это удобно при выездной, мобильной и полевой работе.

Поддерживаются два backend:

- **Serial** — прямое подключение к GPS-приемнику через последовательный порт
  (USB-последовательный адаптер, встроенный COM-порт или `/dev/ttyUSB0`).
- **GPSD** — подключение к серверу [gpsd](https://gpsd.io/) по TCP
  (по умолчанию `127.0.0.1:2947`). Удобно, если GPS используется несколькими
  приложениями или доступен по сети.

Индикатор GPS в строке состояния показывает:

| Цвет | Значение |
|--------|---------|
| Красный `GPS` | Нет подключения / ошибка |
| Желтый `GPS` | Подключено, координаты еще не определены |
| Белый `GPS` | Координаты определены, позиция зафиксирована |

После определения координат grid locator станции заменяется позицией,
рассчитанной по GPS, и в строке состояния добавляется отметка `(GPS)`:

```
Rig SSB - FTDx10/Dipole  ·  Grid JO62TJ43PL (GPS)
```

Включите **Grid from GPS** в настройках Station & Logbook, чтобы использовать
grid GPS при регистрации QSO, передаче APRS-маяков, отображении карты dashboard
и расчете расстояний.

**Grid precision** — настраивается в меню Integration и может составлять 10, 8
или 6 символов. Значение по умолчанию — 10 символов, что соответствует точности
около 25 м. Внутри программы grid всегда рассчитывается с полной точностью, а
при использовании сокращается до заданной длины.

<a id="dx-cluster"></a>
### DX Cluster

Интеграция с DX Cluster использует telnet и требует доступа к интернету.

Сервер по умолчанию:

```text
dxspots.com:7300
```

Доступны фильтры по:

- диапазону,
- континенту споттера,
- виду связи,
- возрасту/времени.

| Клавиша | Действие |
|---|---|
| Enter | Заполнить форму QSO, настроить трансивер и вернуться к QSO |
| Space | Настроить трансивер и остаться в DX Cluster |
| Backspace | Очистить фильтры |

При подключении к DX Cluster в форме QSO появляются две дополнительные возможности:

- **Send a spot** — при заполненной форме нажмите **Ctrl+S**, чтобы открыть диалог и отправить DX spot в кластер.
- **Nearest spots** — при настройке на частоту в форме QSO отображаются до трех ближайших спотов. Это позволяет видеть активность на диапазоне, не покидая экран регистрации. Нажмите **Ctrl+P**, чтобы подставить позывной из ближайшего спота.

<a id="psk-reporter"></a>
### PSK Reporter

Интеграция с PSK Reporter требует доступа к интернету. Это удобный инструмент для быстрой проверки реального прохождения: можно увидеть, кто принимает ваш сигнал или кого принимаете вы на любом диапазоне в данный момент.

Доступны:

- споты прохождения,
- фильтры по диапазону, времени и виду связи,
- карта мира ASCII на **F5**.

<a id="aprs"></a>
### APRS

CQOps поддерживает три типа сервиса APRS. Выберите вариант, соответствующий конфигурации вашей станции:

| Сервис | Подключение | Требуется интернет |
|---|---|---|
| **APRS-IS** | TCP к серверу APRS-IS | Да |
| **KISS** | Последовательный порт к аппаратному KISS TNC | Нет |
| **KISS Server** | TCP к серверу KISS TNC, например Dire Wolf | Нет, при работе в локальной сети |

Тип сервиса выбирается в меню Integrations:

```text
F9 → Integrations → APRS → Service (Space to cycle)
```

Все три сервиса поддерживают прием отчетов о местоположении APRS от ближайших
станций и отображение их на локальной карте CQOps Live с помощью:

- стандартных символов APRS,
- всплывающих окон с позывными,
- автоматического масштабирования,
- настраиваемого круга дальности.

Все сервисы также поддерживают **периодическую передачу координатного маяка**.
CQOps передает grid locator станции с заданным интервалом. Если GPS активен и
включен параметр **Grid from GPS**, маяк автоматически использует координаты
GPS — это удобно при выездной и мобильной работе.

<a id="aprs-is"></a>
#### APRS-IS

Подключается к глобальной сети APRS-IS через интернет. Требуются:

- действительный радиолюбительский позывной,
- passcode APRS-IS, рассчитанный по позывному,
- подключение к интернету.

Сервер по умолчанию:

```text
euro.aprs2.net:14580
```

APRS-IS настраивается глобально в **F9 → Integrations → APRS**.
Позывной, SSID, символ, комментарий, интервал маяка и фильтр дальности для
отдельного журнала задаются в **F9 → Logbooks → [active logbook] → APRS**.

<a id="kiss-serial"></a>
#### KISS (serial)

Подключается непосредственно к аппаратному KISS TNC через последовательный порт.
Интернет не требуется: кадры APRS передаются и принимаются через радиостанцию.

В меню Integrations настройте последовательный порт, baud rate, data bits,
parity, stop bits и DTR/RTS:

```text
F9 → Integrations → APRS → Service: KISS
```

После выбора KISS становятся видимыми поля последовательного интерфейса:
Port, Baud, Data bits, Parity, Stop bits, DTR и RTS.

Кнопка **Test** открывает последовательный порт и проверяет доступность TNC.

<a id="kiss-server-tcp"></a>
#### KISS Server (TCP)

Подключается к KISS TNC, доступному по TCP, например к экземпляру
[Dire Wolf](https://github.com/wb2osz/direwolf), работающему на том же компьютере
или в локальной сети. Подключение к интернету не требуется.

Укажите host и port в меню Integrations:

```text
F9 → Integrations → APRS → Service: KISS Server → Host / Port
```

Значение по умолчанию: `127.0.0.1:8001`

<a id="beaconing"></a>
#### Передача маяка

Маяки передаются с интервалом, заданным отдельно для каждого аппаратного журнала.
Минимальный интервал составляет 1 минуту. Маяк содержит:

- позывной станции с SSID,
- grid locator, полученный по GPS, если он доступен,
- символ APRS,
- необязательный комментарий.

Если **GPS** активен и в настройках Station включен параметр **Grid from GPS**,
маяк автоматически использует grid locator, полученный по GPS. При движении не
требуется вручную обновлять grid.

Интервал маяка и другие параметры отдельного аппаратного журнала настраиваются в:

```text
F9 → Logbooks → [active logbook] → APRS
```

<a id="receiving"></a>
#### Прием

Принятые отчеты APRS о местоположении кэшируются локально и отображаются на карте
CQOps Live dashboard. Станции показываются с символами APRS; по ним можно щелкнуть,
чтобы просмотреть подробности. Масштаб автоматически подбирается так, чтобы показать
все видимые станции в пределах настроенной дальности.

Прием APRS не зависит от передачи маяка: можно принимать без передачи и наоборот.
Достаточно включить APRS в меню Integrations и выбрать тип сервиса.

<a id="solar-data"></a>
### Солнечные данные

Солнечные данные поступают с hamqsl.com и включают:

- SFI,
- число солнечных пятен,
- индексы A/K,
- условия по отдельным диапазонам.

Для обновлений в реальном времени требуется интернет. После успешной загрузки кэшированные данные остаются доступными автономно.

<a id="cqops-live-dashboard"></a>
## CQOps Live Dashboard

CQOps Live — встроенный браузерный dashboard для отображения активности радиостанции в реальном времени.

Он полезен для:

- публичных экранов во время field day,
- экранов клубной радиостанции,
- наблюдения за соревнованием,
- контроля станции из другого помещения,
- стендов на мероприятиях и выставках.

<a id="enable-the-dashboard"></a>
### Включение dashboard

1. Нажмите **F9**.
2. Откройте **Integrations**.
3. Перейдите в **HTTP Server**.
4. Включите **HTTP server**.
5. При необходимости задайте адрес и порт.
6. Нажмите **Ctrl+S**, чтобы сохранить.
7. Откройте dashboard в браузере.

Параметры по умолчанию:

| Параметр | Значение по умолчанию |
|---|---|
| Address | `0.0.0.0` |
| Port | `8073` |
| Local URL | `http://localhost:8073` |

Сервер запускается сразу после сохранения.

> **Привязка адреса:** значение `0.0.0.0` по умолчанию делает dashboard доступным с любого устройства в локальной сети. Это удобно для экранов field day, клубной радиостанции или контроля станции из другого помещения. Чтобы разрешить доступ только с локального компьютера, установите Address в `127.0.0.1`.

<a id="display-modes"></a>
### Режимы отображения

CQOps Live имеет два режима отображения.

<a id="overview-mode"></a>
#### Режим Overview

Отображается, когда активная работа с позывным не ведется.

В нем показываются:

- **карты в реальном времени** — маркеры сегодняшних QSO с дугами большого круга от grid станции к каждому корреспонденту, а также локальная карта APRS со станциями поблизости,
- таблица Recent QSOs,
- сведения о станции,
- статистика,
- темп за 5 минут, 15 минут и 1 час,
- наиболее активные операторы,
- QSO на наибольшем расстоянии.

<a id="active--now-working-mode"></a>
#### Режим Active / Now Working

Отображается во время работы с позывным.

В нем показываются:

- крупный позывной,
- индикатор подвида,
- фотография QRZ, если доступна,
- индикаторы диапазона и вида связи,
- индикаторы DUPE / NEW CALL / NEW DXCC,
- расстояние и азимут,
- выделенная пунктирная трасса на карте от grid станции к grid корреспондента.

<a id="info-box"></a>
### Info box

Info box над локальной картой каждые 5 секунд переключается между модулями:

- условия на диапазонах,
- солнечная активность,
- геомагнитное поле,
- последний spot DX Cluster,
- число отчетов PSK Reporter по диапазонам.

<a id="weather-row"></a>
### Строка погоды

Строка погоды показывает текущие данные Open-Meteo для grid locator станции:

- температуру,
- ветер,
- влажность,
- значок.

Погодные данные загружаются браузером; при отсутствии сети функция корректно становится недоступной.

<a id="local-map"></a>
### Локальная карта

Локальная карта справа предназначена для **наблюдения за ближайшими станциями APRS**. На ней могут отображаться:

- ближайшие станции APRS со стандартными символами,
- всплывающие окна с позывными при наведении или щелчке,
- настраиваемый круг дальности,
- необязательная граница дня и ночи,
- необязательное наложение метеорадара RainViewer.

<a id="real-time-updates-and-performance"></a>
### Обновления в реальном времени и производительность

CQOps Live обновляется через Server-Sent Events (SSE). Обновлять страницу не требуется.

Dashboard рассчитан на маломощное оборудование:

- браузер выполняет отрисовку карт,
- браузер рассчитывает расстояния,
- браузер рассчитывает статистику,
- CQOps передает небольшие обновления JSON,
- если HTTP server отключен, порт не открывается и goroutines dashboard не запускаются.

<a id="dashboard-customization"></a>
### Настройка dashboard

В форме интеграции HTTP Server можно настроить:

| Поле | Описание |
|---|---|
| Header 1 | Основной заголовок в верхней части и главном блоке страницы. Если поле пустое, используется “CQOps Live”. |
| Header 2 | Подзаголовок под основным заголовком. Если поле пустое, используется “Fast, portable ham radio logger”. |
| Logo URL | URL общедоступного изображения в левом верхнем углу. Если поле пустое, используется логотип CQOps. |
| Event Start | Дата в формате `YYYY-MM-DD`. Фильтрует статистику и списки QSO, начиная с этой даты. |

<a id="configuration"></a>
## Конфигурация

Откройте конфигурацию клавишей **F9**.

<a id="configuration-files"></a>
### Файлы конфигурации

| Платформа | Путь к конфигурации |
|---|---|
| Linux / macOS | `~/.config/cqops/config.yaml` |
| Windows | `%APPDATA%\cqops\config.yaml` |

Конфиденциальные учетные данные хранятся отдельно в файле `secrets.enc` в том же каталоге конфигурации.

Secrets шифруются ключом, привязанным к конкретному компьютеру. При переносе конфигурации на другой компьютер учетные данные необходимо ввести заново.

<a id="configuration-menus"></a>
### Меню конфигурации

Нажмите **F9**, чтобы открыть главное меню, затем выберите:

| Меню | Что настраивается |
|---|---|
| General | Единицы измерения, часовой пояс, карта/фотография корреспондента, панель Solar, источники данных SCP/REF, графика Kitty, debug mode |
| Logbooks | Позывной станции, grid, референции, зоны CQ/ITU, регион IARU, grid GPS; Wavelog для каждого журнала (URL, API key, station profile); APRS для каждого журнала (позывной, символ, маяк, дальность) |
| Operators | Профили позывного и имени оператора для станций с несколькими операторами |
| Rigs | Presets трансиверов: модель, антенна, мощность, backend (None/flrig/Hamlib), поворотное устройство, UDP WSJT-X |
| Contests | Профили соревнований: название, дата, ID соревнования ADIF, шаблоны контрольных номеров, начальный порядковый номер |
| Integration | DX Cluster, HTTP server для dashboard, GPS, APRS, Solar, PSK Reporter |
| Callbook | Провайдеры QRZ.com, HamQTH, QRZ.RU и Callook.info; порядок приоритетов, base-call fallback, поиск Wavelog |
| Notifications | Уведомления о сохранении QSO, состоянии отправки QSO в Wavelog, звуковой сигнал дубликата, звуки ошибок |

<a id="multi-logbook"></a>
### Несколько аппаратных журналов

Используйте отдельные аппаратные журналы для домашней, выездной, соревновательной и клубной работы.

Нажмите **Ctrl+L**, чтобы переключить активный аппаратный журнал.

Каждый журнал хранит собственные:

- сведения о станции,
- параметры Wavelog,
- настройки соревнований,
- настройки операторов.

<a id="multi-operator"></a>
### Несколько операторов

Профиль оператора содержит:

- позывной оператора,
- имя оператора.

Нажмите **Ctrl+O**, чтобы переключить активного оператора.

Активный оператор сохраняется в поле ADIF `OPERATOR` и передается при отправке данных в Wavelog.

<a id="multi-rig"></a>
### Несколько трансиверов

Presets трансиверов хранят:

- backend,
- модель,
- антенну,
- мощность,
- параметры поворотного устройства,
- параметры WSJT-X.

Нажмите **Ctrl+R**, чтобы переключить активный трансивер.

<a id="encrypted-secrets"></a>
### Зашифрованные secrets

Начиная с версии v0.8.7 учетные данные хранятся в зашифрованном виде.

| Элемент | Значение |
|---|---|
| Файл secrets | `secrets.enc` |
| Расположение | Тот же каталог, что и `config.yaml` |
| Права Unix | `0600`, если поддерживается |
| Шифрование | AES-256-GCM с ключом, привязанным к компьютеру |
| Защищенные данные | Пароль QRZ, логин DX Cluster, API keys Wavelog |

Secrets в открытом виде из старых конфигураций переносятся при первом запуске.

Если файл `secrets.enc` поврежден, CQOps запускается с предупреждением и предлагает повторно ввести учетные данные.

<a id="troubleshooting"></a>
## Устранение неполадок

<a id="cqops-does-not-start"></a>
### CQOps не запускается

Проверьте:

- размер терминала должен быть не менее 80×24,
- пользователям Windows рекомендуется использовать Windows Terminal,
- запуск может блокироваться сетевой инициализацией; попробуйте:

  ```bash
  cqops --offline
  ```

Проверьте журналы:

| Платформа | Путь к журналам |
|---|---|
| Linux | `~/.local/share/cqops/logs/` |
| macOS | `~/Library/Application Support/cqops/logs/` |
| Windows | `%APPDATA%\cqops\logs\` |

<a id="rig-does-not-connect"></a>
### Трансивер не подключается

Для flrig:

- убедитесь, что flrig запущен,
- проверьте порт в активном preset трансивера,
- порт по умолчанию — `12345`.

Для Hamlib:

- убедитесь, что `rigctld` запущен,
- проверьте host и port,
- убедитесь, что трансивер/backend поддерживает запрашиваемые данные.

Метки состояния помогают определить проблему:

| Цвет | Значение |
|---|---|
| Белый/по умолчанию | Подключено |
| Желтый | Отключено или выполняется подключение |
| Красный | Ошибка |

Всплывающие уведомления о повторном подключении могут подавляться. CQOps может повторять попытки без уведомлений.

<a id="wsjt-x-does-not-auto-log"></a>
### WSJT-X не регистрирует QSO автоматически

Проверьте:

- **Settings → Reporting → UDP Server** в WSJT-X,
- совпадают ли host и port UDP с активным preset трансивера в CQOps,
- используется ли WSJT-X 2.6 или новее,
- активна ли метка состояния WSJT,
- правильно ли выбран активный аппаратный журнал,
- правильно ли выбран активный оператор.

<a id="wavelog-upload-fails"></a>
### Ошибка отправки в Wavelog

Проверьте:

- URL Wavelog,
- API key,
- station profile ID,
- метку состояния **WL**.

Ошибки отправки отображаются как всплывающие уведомления. QSO остаются локально сохраненными даже при неудачной отправке. Ошибка отдельного QSO не блокирует отправку остальных записей пакета.

<a id="config-file-issues"></a>
### Проблемы с файлом конфигурации

Файл конфигурации:

| Платформа | Путь |
|---|---|
| Linux / macOS | `~/.config/cqops/config.yaml` |
| Windows | `%APPDATA%\cqops\config.yaml` |

Файл secrets:

```text
secrets.enc
```

Файл secrets хранится в том же каталоге, что и `config.yaml`.

Если конфигурация повреждена, переместите или удалите ее и перезапустите CQOps. Мастер настройки создаст новую конфигурацию.

Поле `last_fetched_id` появляется только после успешной загрузки из Wavelog.

<a id="performance-issues"></a>
### Проблемы производительности

Попробуйте:

- отключить отрисовку карты в настройках General,
- отключить панель Solar, если она не нужна,
- при отсутствии сети избегать экранов с интенсивным сетевым обменом, таких как DX Cluster и PSK Reporter,
- использовать `cqops --offline` при нестабильной сети.

<a id="reporting-bugs"></a>
## Сообщение об ошибках

Перед созданием сообщения об ошибке:

1. Включите **Debug mode** в **F9 → General → Debug** или задайте:

   ```yaml
   debug: true
   ```

   в файле `config.yaml`.

2. Воспроизведите проблему.
3. Приложите соответствующий журнал.

Сообщайте об ошибках на GitHub:

<https://github.com/szporwolik/cqops/issues>

Укажите:

- версию CQOps из `cqops --version`,
- операционную систему,
- эмулятор терминала,
- шаги для воспроизведения,
- соответствующий debug log.
